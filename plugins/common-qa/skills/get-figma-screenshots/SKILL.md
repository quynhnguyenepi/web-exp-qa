# Export Figma Screenshots

Export high-resolution screenshots from Figma files using the Figma REST API with automatic metadata generation.

## When to Use

Invoke this skill when you need to:

- Export screenshots from Figma design files for QA comparison
- Capture Figma frames as PNG images with metadata
- Filter and export specific pages from a Figma file
- Generate YAML metadata with node IDs and URLs for downstream skills

## Workflow Overview

```
Pre-Flight -> Gather Info -> Locate Script -> Run Export -> Report Results
```

## Execution Workflow

### Step 0: Gather Required Information

Use AskUserQuestion if any of these are missing:
- **Figma URL**: Full URL to the Figma file (e.g., `https://www.figma.com/design/FILE_KEY/NAME`)
- **Personal Access Token (PAT)**: Can be provided via argument or `FIGMA_PAT` environment variable
- **Page Filter** (optional): Specific page name to export from (e.g., "• Axiom 1 [MVP]")
- **Output Directory**: Where to save screenshots and metadata

### Step 1: Locate the Export Script

Use Glob to find the script (works from any repo via plugin cache):
```
Glob("**/get-figma-screenshots/scripts/export_figma_screenshots.py")
```
This searches both the plugin cache (`~/.claude/plugins/cache/`) and any local plugin directories.

### Step 2: Run the Export Script

Execute the bundled script with appropriate parameters:

**Basic usage:**
```bash
export FIGMA_PAT="figd_..."
python3 <path-to-script>/export_figma_screenshots.py \
    --url "https://www.figma.com/design/FILE_KEY/NAME" \
    --output "path/to/output"
```

**With page filter:**
```bash
python3 <path-to-script>/export_figma_screenshots.py \
    --url "https://www.figma.com/design/FILE_KEY/NAME" \
    --page "Page Name" \
    --output "path/to/output"
```

**All parameters:**
- `--url`: Figma file URL (required)
- `--pat`: Figma Personal Access Token (optional if FIGMA_PAT env var set)
- `--page`: Filter to specific page name (optional)
- `--output`: Output directory path (required)
- `--scale`: Export scale, default 2 for high resolution (optional)
- `--depth`: Frame depth level to export, default 3 for top-level screens (optional)

### 4. Verify Output

The script generates:
- **PNG screenshots**: High-resolution (2x) images with safe filenames
- **YAML metadata files**: One per screenshot with:
  - `frame_number`: Sequential number
  - `screenshot_file`: PNG filename
  - `figma_file_url`: Source Figma file URL
  - `figma_node_id`: Exact node ID from Figma
  - `figma_node_url`: Direct link to node in Figma
  - `node_name`: Frame name from Figma
  - `page`: Page name containing the frame
  - `frame_type`: Prefix from frame name (e.g., "fx" from "fx/fields")
  - `description`: Human-readable description
- **file_structure.json**: Complete Figma file structure for reference
- **README.md**: Auto-generated documentation of all exported frames

## Understanding Figma File Structure

Figma files have predictable depth levels:

- **Depth 0**: Document (root)
- **Depth 1**: Canvas (pages in Figma)
- **Depth 2**: Section (organizational grouping)
- **Depth 3**: Frame (top-level screens and components)

Default depth of 3 exports top-level frames, which are typically the screens/components you want to document.

## Output Organization

Exported files are organized flat in the output directory:

```
output-directory/
├── file_structure.json              # Complete Figma file structure
├── README.md                         # Auto-generated documentation
├── frame_name_1.png                  # Screenshot
├── frame_name_1.yaml                 # Metadata
├── frame_name_2.png
├── frame_name_2.yaml
└── ...
```

Safe filenames are generated by replacing slashes and spaces with underscores.

## Examples

### Example 1: Export All Frames from File

User: "Export all screenshots from this Figma file: https://www.figma.com/design/ABC123/MyProject"

Response:
1. Ask for PAT if not in environment
2. Run script with minimal parameters:
   ```bash
   export FIGMA_PAT="figd_..."
   python3 <path-to-script>/export_figma_screenshots.py \
       --url "https://www.figma.com/design/ABC123/MyProject" \
       --output "screenshots"
   ```
3. Report results: "Exported 45 frames to screenshots/"

### Example 2: Export Specific Page

User: "Export screenshots from the 'MVP Screens' page only"

Response:
1. Ensure Figma URL and PAT are available
2. Run script with page filter:
   ```bash
   python3 <path-to-script>/export_figma_screenshots.py \
       --url "https://www.figma.com/design/ABC123/MyProject" \
       --page "MVP Screens" \
       --output "mvp-screenshots"
   ```
3. Report: "Exported 12 frames from 'MVP Screens' page"

### Example 3: Custom Scale and Depth

User: "Export at 3x scale and grab frames at depth 4"

Response:
```bash
python3 <path-to-script>/export_figma_screenshots.py \
    --url "https://www.figma.com/design/ABC123/MyProject" \
    --scale 3 \
    --depth 4 \
    --output "high-res-exports"
```
