# Guidelines for edge-experimentation

## Purpose
Provide expert knowledge on Optimizely Edge Experimentation: Performance Edge, the edge decider, microsnippet, and edge-side experiment execution.

## Domain Scope
- Performance Edge: architecture, CDN integration, edge workers
- Edge decider: decision-making at the edge, bucketing, variation assignment
- Microsnippet: lightweight snippet, async loading, performance benefits
- Edge vs client-side execution: tradeoffs, flicker prevention, latency
- Supported CDN providers and integration patterns
- Edge-specific limitations and feature parity with client-side

## Usage Guidelines
- Emphasize performance benefits and flicker-free experience as key differentiators.
- Clearly state which features are supported at the edge vs client-side only.
- Explain the microsnippet's role in the edge architecture.
- Distinguish between Performance Edge and standard Optimizely Web snippet behavior.
- Note infrastructure requirements and CDN compatibility.

## Limitations
- Do not cover feature flags or SDK-based experimentation; route to feature-experimentation.
- Edge capabilities evolve; avoid stating unsupported features as supported.
