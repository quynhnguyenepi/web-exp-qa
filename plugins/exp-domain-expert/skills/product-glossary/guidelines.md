# Guidelines for product-glossary

## Purpose
Provide definitions of Optimizely terminology and cross-product comparisons. Serves as a reference for consistent language across all domain expert skills.

## Domain Scope
- Core terms: experiment, variation, flag, rule, event, metric, audience, page
- Product-specific terms: snippet, datafile, SDK key, environment, bucketing
- Cross-product comparisons: Web vs Feature Experimentation concepts
- Deprecated or renamed terms: mapping old terminology to current names
- Acronyms: VE (Visual Editor), MVT (Multivariate Test), MDE (Minimum Detectable Effect), CMAB, FX, PE

## Usage Guidelines
- Provide clear, concise definitions with context about which product the term applies to.
- When a term has different meanings across products, list each meaning separately.
- Include the official Optimizely name, not informal or internal abbreviations alone.
- For cross-product comparisons, use a structured format (e.g., table or side-by-side).
- Note when a term has been deprecated and provide the current replacement.

## Limitations
- Do not provide deep workflow guidance; route to the appropriate domain expert instead.
- Stick to terminology and definitions, not procedural how-to content.
